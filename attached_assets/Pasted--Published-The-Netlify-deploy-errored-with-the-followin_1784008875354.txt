Diagnosis

Error type: runtime configuration error during Vite config evaluation.
Cause: vite.config.ts requires the PORT environment variable at config-load time and throws when it is absent. The build runs in Netlify’s build environment where PORT is not set, so Vite fails to load the config and the build aborts (line 71, line 73).
Solution You have two safe fixes; pick one (recommended: change the config so build does not require PORT).

Preferred — make vite.config.ts tolerant of missing PORT
Edit artifacts/mockup-sandbox/vite.config.ts to supply a default or only require PORT for dev. For example replace the code that throws when PORT is missing with a fallback:
// artifacts/mockup-sandbox/vite.config.ts
// Example: use process.env.PORT if present, otherwise fall back to a default port or undefined
const port = process.env.PORT ?? '5173'; // choose any sensible default or undefined for production builds

export default defineConfig({
  // ... other config
  server: {
    port: Number(port),
    // only include server options in dev; or guard with process.env.NODE_ENV !== 'production'
  }
});
Or ensure you only read/require PORT for dev:
const serverOptions = process.env.NODE_ENV !== 'production' ? { port: Number(process.env.PORT ?? 5173) } : undefined;

export default defineConfig({
  // ...
  server: serverOptions
});
Commit this change and re-run the Netlify build. This ensures the Vite config can be evaluated during the build phase without a PORT env var.
Alternative — provide PORT in Netlify build environment (less ideal)
In the Netlify UI: Site → Site settings → Build & deploy → Environment → Environment variables, add a variable:
Key: PORT
Value: 5173 (or a port your config expects)
Re-run the build. This avoids the thrown error because PORT will now be present.
Notes

Prefer option 1 because builds on CI should not depend on a dev-only environment variable. The core problem is the vite config throwing when PORT is not set; modifying the config to default or only use PORT in dev fixes it permanently.
