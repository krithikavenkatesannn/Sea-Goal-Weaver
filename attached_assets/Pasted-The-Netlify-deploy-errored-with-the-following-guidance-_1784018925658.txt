The Netlify deploy errored, with the following guidance provided:

Diagnosis
- Relevant log lines: the build runs the command defined in netlify.toml ([line 67](#L67), [line 85](#L85)) and fails when trying to install pnpm globally: [lines 69–74](#L69-L74) show npm error EEXIST (file already exists). The overall command fails: [line 81](#L81).
- Error type and cause: npm EEXIST when running `npm install -g pnpm` — a global pnpm executable already exists in the build environment at bin/pnpm, so npm refuses to write a file that already exists. Netlify’s image already provides pnpm (or a leftover symlink exists), so attempting to install it globally causes the failure.

Solution
1. Preferred fix — stop reinstalling pnpm in the build:
   - Edit netlify.toml in the repo (file: netlify.toml) — remove the `npm install -g pnpm &&` prefix from the build command and let the environment’s pnpm be used. For example change the build.command to:
```toml
[build]
  command = "pnpm install && pnpm --filter @workspace/ocean-goals run build:netlify"
  publish = "artifacts/ocean-goals/dist/public"
```
   - Commit and push the change. Netlify will then use the already-installed pnpm and the EEXIST error will be avoided.

   Link to the file in your repo: https://github.com/krithikavenkatesannn/Sea-Goal-Weaver/blob/main/netlify.toml

2. Alternative quick fixes (less recommended):
   - Overwrite the existing pnpm by forcing the install (not recommended):
```bash
npm install -g pnpm --force && pnpm install && pnpm --filter @workspace/ocean-goals run build:netlify
```
   - Or ignore install failure so the rest of the command still runs:
```bash
npm install -g pnpm || true && pnpm install && pnpm --filter @workspace/ocean-goals run build:netlify
```
   These can be applied by updating build.command in netlify.toml, but they mask the underlying issue and are not ideal.

Why the preferred fix is better
- Netlify’s build image typically already provides pnpm (or Corepack can provide it). Reinstalling global CLIs can conflict with the environment and is unnecessary. Removing the redundant installation step is the cleanest solution.

If you want to explicitly manage pnpm versions via Corepack instead, you can run (optional):
```bash
corepack enable
corepack prepare pnpm@<version> --activate
pnpm install
...
```
No Node version change is required here.
